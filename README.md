#  SEDMS: Space-Efficient Dynamic Metric Selection for Streaming Telemetry in Edge Networks

## Overview
Edge networks increasingly depend on streaming telemetry to observe traffic generated by heterogeneous IoT devices. Yet, continuously processing and uploading fine-grained telemetry metrics from per-packet streams at the edge is difficult under tight constraints on memory, computation, and bandwidth. In addition, the importance of telemetry metrics may drift with changing traffic patterns due to workload changes or attack threats, making static metric configurations suboptimal in dynamic environments.

We propose SEDMS, a Space-Efficient framework for Dynamic Metric Selection. SEDMS enables edge gateways to maintain a compact and informative metric subset in an online manner without storing raw traffic or repeatedly retraining models. It combines (i) a bounded-memory matrix sketch to summarize high-volume telemetry data, (ii) a spectral-sparse optimization that performs metric selection directly on the sketch, and (iii) a lightweight dynamic update strategy that captures new key metrics by ensuring the approximation of sketch-saved information and triggers metric selection.

## Datasets
We evaluate SEDMS on four datasets derived from public network traffic benchmarks.All packets are featurized with D=115 telemetry metrics using the [Kitsune feature extractor] (https://github.com/ymirsky/Kitsune-py)
N-BaIoT dataset: https://archive.ics.uci.edu/dataset/442/detection+of+iot+botnet+attacks+n+baiot
Kitsune dataset: https://www.kaggle.com/datasets/ymirsky/network-attack-dataset-kitsune
